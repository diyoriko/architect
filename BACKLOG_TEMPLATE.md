## Правила

### Нумерация
- Префикс проекта: `SAMI-`, `HUNT-`, `VT-`, `PORT-`, `ARCH-`
- Номер — последовательный, не переиспользуется
- "Следующий номер" обновляется при каждом добавлении

### Жизненный цикл
1. **Sprint** — набор задач на текущую итерацию
2. **Реализация** — код, тесты
3. **Деплой** — git push + version bump (vX.Y.Z)
4. **Архивация** — спринт сворачивается в `<details>` под номером релиза
5. **Перенос** — незакрытые задачи переносятся в следующий спринт с причиной

### Приоритеты
- **P0** — блокер, делать сейчас
- **P1** — важно, в текущем спринте
- **P2** — улучшение, в следующем спринте
- **P3** — nice-to-have, когда будет время

### Эпики
Группировка по теме, не по приоритету. Примеры:
- `INFRA` — инфраструктура, CI/CD, деплой
- `UX` — пользовательский опыт
- `GROWTH` — рост, маркетинг, дистрибуция
- `TECH` — техдолг, рефакторинг
- `CONTENT` — контент, тексты, SEO

### Источники задач
- Стратег (ежедневно/еженедельно) → автоматически через extract-tasks
- Mega Reviewer (еженедельно) → автоматически через sync_items
- CodeRabbit (PR review) → через coderabbit-to-backlog.yml
- Ручные — от владельца проекта

### Форматирование
```markdown
- [ ] **PROJ-NNN** Краткое название — детальное описание
- [x] **PROJ-NNN** Сделанная задача — описание (дата завершения)
```
